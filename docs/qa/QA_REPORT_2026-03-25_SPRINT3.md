## Issues Found

1. `predicted_margin` absent from live predictor output — Severity: High

   `get_today_predictions()` returns prediction dicts that contain all quarter fields and both score fields but not `predicted_margin`. The field must be computed and inserted in the predictor (`hybrid_team_player.py` or wherever `get_today_predictions` assembles its output) so that the tracker can log it. Without it, margin tracking is a permanent blind spot in `prediction_tracking.csv`.

   Recommended fix: In the function that assembles the prediction dict for export/display, add `prediction['predicted_margin'] = prediction['predicted_home_score'] - prediction['predicted_away_score']` if not already computed upstream.

2. Home quarter sum vs final score delta is 6 pts for today's first prediction — Severity: Low (advisory)

   The delta is within the <10 threshold but is non-trivial. Monitor across the full slate. If the majority of games show home deltas in the 5–9 range, the home-side quarter league averages should be recalibrated against the full-game league average (114.15 → 4 * ~28.5 = 114 — effectively in agreement, so the delta is likely rounding in the per-quarter integer rounding step).

3. Q4 model has zero predictive value (R² = 0.000, coefficient ≈ 0) — Severity: Medium (advisory)

   This is a pre-existing known gap from the analyst's backtest report. Not a new finding from this QA run, but recorded here for the pipeline record. The quarter model needs retraining on 2,000+ games across multiple non-bubble seasons before Q4 (and Q2/Q3) predictions carry meaningful signal.

4. `grep -c " -[0-9]"` regex is too broad for spread-notation detection — Severity: Low

   Recommend updating the Check 6 test to use a pattern that requires a team abbreviation prefix before the negative number.
